# Build snapshot — Run 001 — 2026-07-09

Baseline recommendation for the bare-metal Linux gaming + native-inference + isolated-worker-VM box.
Anchored on the Tweakers high-end AM5 game-pc tier, Zen 5 X3D, and the July-2026 ROCm/Linux state.
Prices indicative NL retail incl. BTW; tighten to exact Pricewatch on next live rerun.

## Recommended configurations

### A. Value / gaming-first — ~€1,600–1,750
9800X3D · RX 9070 XT 16GB · B850 · 2×32 DDR5-6000 CL30 · 850W · big air · North.
Lowest idle (single CCD + RDNA4), best €/gaming-frame. 8 cores is the only compromise — fine
unless you routinely game *while* the host is under heavy VM load.

### B. Do-everything (recommended default) — ~€2,050–2,300
9950X3D · RX 9070 XT 16GB · X870(E) · 2×32 DDR5-6000 CL30 · 850W · big air · North (XL if clearance).
16c/32t absorb games + native local inference + the isolated Claude-worker VM concurrently; X3D CCD keeps
gaming top-tier. This is the coherent pick for the three-role brief. Idle a touch higher than A.

### C. Local-AI-max variant — B, but GPU → RX 7900 XTX 24GB
Take config B and swap the GPU. Choose this only if you want real local-LLM headroom now (24GB,
most mature ROCm). Costs you idle efficiency (RDNA3 draws more at idle — real 24/7 cost) and it's
last-gen on used-market pricing. If the AI-VM ambition is bigger still, the **Radeon AI PRO R9700
32GB (RDNA4)** is the high-VRAM escape hatch without going Strix Halo.

## Component notes / rationale

**CPU.** All Ryzen 9000 desktop chips keep a 2-CU RDNA2 iGPU — essential here, because it lets the
host run headless on the iGPU while the discrete GPU passes through to the gaming VM. Fork is
16-core (9950X3D) vs 8-core (9800X3D): more cores for the concurrent host/VM load vs lower idle and
lower price. The 9850X3D (~3% faster gaming) isn't worth the premium; the 9950X3D2 Dual Edition
(cache on both CCDs, ~$899-class) is a halo part, mention only if cross-CCD latency ever bites.

**GPU.** For AMD+Linux in mid-2026: RDNA4 (RX 9070 XT) is now officially in ROCm 7.2 and idles
lower — the better always-on citizen — but 16GB caps local models to ~20B dense / ~30B MoE at Q4,
and there's a known gfx1201 rocBLASLt lookup bug (workaround: AMD's ROCm Docker image, or
llama.cpp built with HIP / Vulkan). RDNA3 (RX 7900 XTX) is the mature-ROCm, 24GB, no-env-hack
option — better for local AI, worse for idle. Since role #2's AI VM is mainly a *remote* agent
client (network + CPU + RAM), the GPU is chosen for gaming with local-LLM as a bonus → 9070 XT is
the default; 7900 XTX / R9700 only if you're deliberately prioritising local inference.

**Motherboard.** Must be 4-DIMM (2 now, 2 free for 128GB later), with clean IOMMU groups for
single-dGPU passthrough and ≥2.5GbE (dual/10GbE preferred for the LAN-isolation firewall-VM
design). Confirm the exact board's IOMMU groups from Phoronix / passthrough reports before buying.

**RAM.** 2×32 DDR5-6000 CL30 EXPO is the AM5/X3D sweet spot (fclk 2000, 1:1). Populating all 4
DIMMs later to hit 128GB **drops the stable speed** (signal integrity) — so for capacity-at-speed
the cleaner future path is 2×48 (96GB) or 2×64 (128GB) modules rather than 4×32. DDR5 pricing is
the volatile line item; this is the single biggest reason the tool is rerunnable.

**PSU / cooling / case.** 850W ATX 3.1 Gold/Platinum for low-load efficiency (and 7900 XTX
transients if config C). Big air cooler (Peerless Assassin 120 SE ≈ €40, or NH-D15) — silent,
no pump wear for 24/7; X3D runs hot but air copes in eco/curve-optimizer. Fractal North for the
aesthetic; North XL if you'd rather not think about GPU length.
